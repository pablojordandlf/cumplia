# CumplIA Web Application - Testing Guide

This document outlines how to run tests for the CumplIA web application.

## Prerequisites

- Node.js and npm/yarn installed
- Project dependencies installed (`npm install` or `yarn install`)

## Setup

1.  **Install Dependencies:**
    If you haven't already, install all project dependencies:
    \`\`\`bash
    npm install
    \`\`\`
    or
    \`\`\`bash
    yarn install
    \`\`\`

2.  **Jest Configuration:**
    The Jest configuration is located at `jest.config.js`. It's pre-configured to work with TypeScript, React, and JSDOM.
    A setup file, `jest.setup.js`, is also included to provide global Jest matchers like `toBeInTheDocument`.

3.  **Test Environment:**
    The test environment is set up to simulate a browser environment using `jsdom`. Mocks for Supabase and `fetch` are configured in the individual test files.

## Running Tests

You can run tests using the `npm test` or `yarn test` command.

### Watch Mode

To run tests in watch mode (which automatically re-runs tests when files change):
\`\`\`bash
npm test
\`\`\`
or
\`\`\`bash
yarn test
\`\`\`

### CI Mode

For running tests in a Continuous Integration environment (non-interactive):
\`\`\`bash
npm run test:ci
\`\`\`
or
\`\`\`bash
yarn test:ci
\`\`\`

## Test Coverage

Test coverage reports can be generated by running Jest with the `--coverage` flag:
\`\`\`bash
npm test -- --coverage
\`\`\`
or
\`\`\`bash
yarn test --coverage
\`\`\`
This will create a `coverage` directory with detailed reports.

## Test File Structure

Test files are located in the `__tests__` directory, mirroring the source code structure:

-   **Unit Tests:**
    -   `__tests__/lib/api/client.test.ts`
    -   `__tests__/lib/api/use-cases.test.ts`
    -   `__tests__/lib/document-generator.test.ts`
-   **Component Tests:**
    -   `__tests__/components/ui/UseCaseCard.test.ts` (if `UseCaseCard` exists)
-   **API Route Tests:**
    -   `__tests__/api/use-cases.test.ts`

## Mocking

-   **Supabase:** The Supabase client is mocked using `jest.mock('@supabase/supabase-js')` in relevant test files.
-   **Fetch API:** For tests involving network requests (like API routes or client-side fetches), `jest-fetch-mock` is assumed to be configured and used.
-   **Next.js API Routes:** Tests for API routes simulate the request and response objects, often using libraries like `next-test-api-route-handler` or by manually mocking.

## Notes

-   Ensure that any components or utility functions you are testing are correctly imported and that their paths are accurate within the test files.
-   If you encounter issues with mocks or imports, verify the file paths and the structure of the code being tested.

## Contributing to Tests

When adding new tests or modifying existing ones, please ensure:
-   All core functionalities are covered.
-   Mocks are used effectively to isolate units under test.
-   Tests are clear, concise, and maintainable.
-   Tests pass reliably in both watch mode and CI mode.
